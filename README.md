# 天市×indeedPLUS運用実績 ローカルParquet版

GitHub Pagesで動かす、ローカルParquetデータセット読み込み版のブラウザアプリです。

## 目的

従来の `data.csv` / `data-list.json` 読み込みではなく、ユーザーがローカルPC上の `indeedマージレポート` または `output_dataset` フォルダを選択し、`manifest.json` と `fact/*.parquet` をDuckDB-Wasmで読み込んで集計します。

## 想定フォルダ構成

```text
indeedマージレポート
└─ output_dataset
   ├─ fact
   │  ├─ fact_202605.parquet
   │  └─ ...
   ├─ logs
   │  └─ build_work.duckdb
   ├─ master
   │  ├─ company_master.parquet
   │  └─ month_master.parquet
   └─ manifest.json
```

アプリで主に使うのは以下です。

```text
output_dataset/manifest.json
output_dataset/fact/fact_YYYYMM.parquet
```

`company_master.parquet` と `logs/build_work.duckdb` はこの版では読み込みません。

## GitHub Pagesへの配置方法

1. このフォルダ内の `index.html` をGitHubリポジトリの公開対象ブランチに配置します。
2. GitHub Pagesを有効化します。
3. 公開URLにアクセスします。
4. 「フォルダを選択」から `indeedマージレポート` または `output_dataset` を選択します。
5. 初期集計が実行されます。

## 初期仕様

- 初期期間：`manifest.json` の `month_max` を最新月とし、最新月を含む直近12か月
- フリーワード検索：`タイトル` を対象
- 勤務地検索：`都道府県 + 市区町村` を対象
- キャンペーン絞り込み：すべて / SAのみ / SA以外
- 職業カテゴリー：削除
- 関連職種：`関連職種` を `|` で分割
- 比較軸：職種比較 / 都道府県比較 / 月別推移 / 期間A-B比較

## 注意点

- ブラウザのセキュリティ仕様上、ローカルフォルダはユーザーが毎回選択する必要があります。
- DuckDB-Wasm本体はCDNから読み込むため、初回表示時にインターネット接続が必要です。
- Parquetファイルはブラウザ内で読み込まれ、外部サーバーへアップロードされる設計ではありません。
- ただし、CDNから読み込むライブラリ自体を使用するため、完全オフライン運用にする場合はDuckDB-Wasmの関連ファイルをリポジトリ内に同梱する方式へ変更してください。
